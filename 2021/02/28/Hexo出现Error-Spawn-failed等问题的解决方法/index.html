Hexo出现Error: Spawn failed等问题的解决方法

写于 2021 年 02 月 28 日 21 时 43 分
更新于 2021 年 02 月 28 日 22 时 28 分

Error: Spawn failed以及fatal: unable to access等

今天尝试hexo d失败，报错为fatal: unable to access ‘https://github.com/…，如下图所示。

一开始看到Fatal to connect to github.com port 443: Time out以为是没用魔法或者网络波动导致的，于是更换网络环境之后，此行报错消失，但仍旧无法deploy成功，并且还要输密码，似乎SSH失效了。并且发现下面还有一个奇怪的错误，显示Error: Spawn failed，看不懂这个到底是什么问题。不过我上次也遇到输密码的情况，但是输入密码之后deploy成功，就一直没管这个事，结果今天直接不能deploy了。

先是上网搜了一下，https://blog.csdn.net/HTL2018/article/details/106876940 中的方法即：

  1. 删除.deploy_git文件夹;
  2. 输入git config --global core.autocrlf false
  3. 然后，依次执行：
    hexo clean hexo g hexo d

该方法的确能够让hexo d成功，虽然最后还是要输一遍密码，但是经再次尝试，hexo d中故意输错一次github密码，出现fatal: Authentication failed for…的报错，同时含有Error: Spawn failed，再次尝试hexo d则回到图中的状态。可见该方法并不能够彻底解决该问题。于是怀疑是SSH哪里出了问题。尝试重新部署SSH。
